# CHRONORIFT_GAME_DESCRIPTION_CHAPTERS_STAGES_BADGES_V1

Status: full game description candidate. No runtime replacement.

## Purpose

Describe ChronoRift as a complete game structure: story chapters, stages, achievements, badges, package logistics, item-driven RPG progression, gizmo participation, PR Worm pressure, and modular construction in a 20 x 20 x 20 workspace.

## Design boundary

External references such as modular construction games, farm-like loops, cinematic puzzle games, or AI-antagonist structures are mood and category references only. ChronoRift must remain original in characters, dialogue, systems, names, assets, levels, and story.

## Complete game frame

ChronoRift is an office RPG and logistics-builder where the player is recruited by a global Corporation as a HUMAN in the LOOP. The player starts with simple package work, then discovers that the work system, supervisor, boss, PR Worm, and Pos.Aya / WayPoint-O are all part of a deeper corporate loop.

The game begins with story, identity, personal items, and Stuff Box. It then expands into cubicle work, package routing, color labeling, modular construction, repair minigames, portal/time systems, and finally a life-outside-the-loop arc.

## Chapter structure

### Chapter 0. Recruitment and identity

Story:
- Corporate ad reaches the player.
- Player applies to become HUMAN in the LOOP.
- Player selects avatar and personal items.
- Player stores items in Stuff Box.
- Closing the locker starts the game.

Stages:
- Stage 0.1: Watch recruitment intro.
- Stage 0.2: Apply for job.
- Stage 0.3: Choose avatar placeholder.
- Stage 0.4: Select personal items.
- Stage 0.5: Store items in Stuff Box.

Achievements:
- First Application: complete job application.
- Identity Seed: choose avatar and name.
- Stuff Box Ready: store all selected personal items.

Badges:
- Early Identity Badge: if the player inspects avatar/personal item details before the prompt.
- Personal Meaning Badge: if the player selects all meaningful personal items before moving on.

### Chapter 1. Cubicle obedience

Story:
- Player starts in cubicle.
- Notebook records story, stage, skills, achievements, and Somero trace.
- Laptop shows supervisor and boss messages.
- Pos.Aya appears as a corporate tool, not yet a revealed folded agent.

Stages:
- Stage 1.1: Open Notebook.
- Stage 1.2: Open Laptop.
- Stage 1.3: Read supervisor first task.
- Stage 1.4: Read boss welcome message.
- Stage 1.5: Stand up from desk and return.

Achievements:
- First Log: open Notebook.
- First Order: read supervisor first email.
- Corporate Welcome: read boss message.
- Cubicle Awareness: stand up and return.
- Gizmo Handshake: open Pos.Aya or receive first hint.

Badges:
- Before Asked Badge: perform a required interaction before the tutorial asks for it.
- Efficient Orientation Badge: complete all cubicle orientation steps under target time.

### Chapter 2. Basic package loop

Story:
- Player learns package receiving and sending.
- Start/New Batch button delivers a new batch of packages.
- Player tests basic movement, push buttons, shipping, and first goals.
- Early rewards teach compliance but also begin repetition.

Stage design:
- Packages arrive only by batch.
- The player uses a Start/New Batch button to receive packages.
- The player moves boxes to shipping area.
- A push button sends every valid package currently in the shipping area.
- Buttons are placed at opposite ends, with wall buttons to the side.
- Portal-capable walls exist above/below and on adjoining walls to enable later routing.

Stages:
- Stage 2.1: Receive first batch.
- Stage 2.2: Transport 5 boxes using basic skills.
- Stage 2.3: Send 5 packages; first stage clear.
- Stage 2.4: Send 5 packages faster by stacking two at a time.
- Stage 2.5: Use gizmo to understand route or package state.
- Stage 2.6: Open or inspect portal-capable surfaces.
- Stage 2.7: Send 10 boxes; stage clear.
- Stage 2.8: Send 20 boxes; stage clear.
- Stage 2.9: Complete a timed shipping challenge before deadline; stage clear.

Achievements:
- First Batch: press Start/New Batch and receive packages.
- First Five: send 5 packages.
- Stack Pair: stack and move two boxes together.
- Fast Five: send 5 packages under target time.
- Gizmo Route: use Pos.Aya to inspect route or package state.
- Portal Awareness: inspect or open a valid portal surface.
- Ten Sent: send 10 boxes.
- Twenty Sent: send 20 boxes.
- Deadline Clear: complete shipment before deadline.

Badges:
- Sequence Break Badge: completing a later shipping count before the tutorial expects it.
- Record Badge: improving a previous shipping time.
- Stack Master Badge: completing a stage primarily by stack pairs.
- Batch Discipline Badge: clearing without losing or misplacing boxes.
- Combo Shipping Badge: earning several shipping achievements in one run.

Shipping area rule:
- The shipping area can accept up to 20 boxes for one send action.
- This corresponds to a 5 x 2 x 2 arrangement.
- A valid 20-box stack can be sent with one push button if positioned correctly.

### Chapter 3. Labeling, scale, and color zones

Story:
- Packages become more complex.
- The player must request scale and labeling equipment.
- Packages are weighed and labeled.
- Label color determines shipping zone.
- The work loop becomes classification, not only transport.

Stages:
- Stage 3.1: Request scale from laptop or supply system.
- Stage 3.2: Request labeling station.
- Stage 3.3: Activate scale and label workflow.
- Stage 3.4: Apply label at completion.
- Stage 3.5: Sort boxes by label color into zones.
- Stage 3.6: Complete mixed-color shipment.

Achievements:
- Scale Requested: successfully request scale.
- Label Station Online: activate label station.
- First Label: apply first package label.
- Color Match: send a package to correct color zone.
- Clean Sort: sort full batch with no incorrect zones.
- Mixed Batch Clear: complete a mixed-label shipment.

Badges:
- Early Sort Badge: correctly place color packages before instruction completes.
- Perfect Label Badge: full batch labeled and sent without error.
- Fast Classification Badge: mixed-color shipment completed under target time.

### Chapter 4. Construction and classification systems

Story:
- Classification opens the need for building systems.
- Player can request cubes and devices to build transport, elevators, slides, and classification pathways.
- The workspace starts becoming modular and player-authored.

Construction access:
- request cubes from laptop / printer / supply system
- place blocks on floor, wall, or ceiling
- build routes from receiving area to shipping zones
- combine conveyors, elevators, slides, ramps, guides, and portal surfaces

Stages:
- Stage 4.1: Request first construction cube.
- Stage 4.2: Place first block.
- Stage 4.3: Build simple conveyor route.
- Stage 4.4: Build color classification branch.
- Stage 4.5: Build elevator-assisted route.
- Stage 4.6: Build slide / toboggan route.
- Stage 4.7: Deliver classified packages through constructed system.

Achievements:
- First Build: place first block.
- Route Builder: construct a path from receiving to shipping.
- Elevator Logic: use elevator block correctly.
- Slide Logic: use slide/toboggan block correctly.
- Classifier: route packages by color.
- Automated Batch: complete shipment using constructed system.

Badges:
- Improvised Engineer Badge: solve with fewer blocks than expected.
- Overbuilt But Works Badge: solve with a large construction that still works.
- Minimal Route Record Badge: beat route with fewer blocks or faster time.
- Early Automation Badge: automate before the tutorial demands it.

### Chapter 5. Jam alerts and repair minigames

Story:
- Constructed systems start failing or jamming.
- Boxes can get stuck.
- Alert lights, alarms, and visible gizmo signals identify jammed areas.
- Clicking/tapping a jammed area opens a contextual action menu.
- Choosing Repair opens a minigame.

Jammed area feedback:
- alarm sound or visual pulse
- warning light
- area highlighted in gizmo vision
- visible signal on map and local view
- action menu: inspect, repair, reroute, clear, enter minigame

Repair minigame types:

#### 5A. Side runner repair

View:
- 2D lateral runner
- scaffolds
- elevators
- falling boxes
- box safety constraints

Rules:
- boxes must not break
- broken boxes reduce coins or score
- player uses jumps, platforms, elevators, and timing tools
- goal is to reach jam location or move repair blocks into place

Achievements:
- First Repair: clear first jam.
- No Damage Repair: clear without breaking boxes.
- Jump Save: prevent a falling box from breaking.
- Elevator Rescue: use elevator in repair minigame.

Badges:
- Fragile Handler Badge: complete repair with no box damage.
- Fast Repair Badge: clear jam under target time.
- Tool Master Badge: use every repair tool correctly in one run.

#### 5B. Toboggan / slide runner repair

View:
- chute or toboggan runner
- multiple route choices
- levers divert packages to review lanes
- tires or bumpers push packages off route
- portals can appear inside runner routes

Rules:
- choose route at forks
- jump or avoid review-diversion levers when necessary
- tires pushing boxes out of lane reset the runner segment
- collect and recompile boxes
- arrange packages into clean stacks
- build a 20-box white batch for clean shipping
- successful run sends stack flying into the shipping counter / shipping area

Achievements:
- Chute Runner Clear: complete slide repair.
- Portal Runner Use: use portal inside runner.
- Lever Avoid: avoid wrong diversion lever.
- Tire Dodge: avoid tire/bump reset.
- Twenty Stack Recovery: rebuild a full 20-box shipment.

Badges:
- Clean Conveyor Badge: no resets in toboggan runner.
- White Stack Badge: build a perfect 20-box white stack.
- Portal Chute Badge: solve runner using portals efficiently.
- Anti-Jam Specialist Badge: several repair achievements in one jam cycle.

### Chapter 6. PR Worm pressure and energy cost

Story:
- PR Worm intensifies messages encouraging endless productivity.
- Supervisor pressures the player to continue.
- Boss reframes exhaustion as opportunity.
- Player and Pos.Aya begin losing energy.
- The player starts questioning the loop.

Stages:
- Stage 6.1: Receive PR Worm productivity nudge.
- Stage 6.2: Continue working despite low energy.
- Stage 6.3: See energy warning.
- Stage 6.4: Use personal item or rest option.
- Stage 6.5: Trigger first meaningful resistance choice.

Achievements:
- First Fatigue: see energy decline.
- Rest Recognized: choose a recovery action.
- Personal Item Return: use an item from Stuff Box.
- Refuse the Loop: choose not to continue one task immediately.

Badges:
- Energy Awareness Badge: recover before critical energy.
- Against the Nudge Badge: ignore PR Worm pressure and recover.
- Meaningful Pause Badge: stop productivity to restore purpose.

### Chapter 7. Folded agent discovery

Story:
- Pos.Aya / WayPoint-O remembers more than expected.
- It logs events the player did not explicitly record.
- It notices item ownership and energy state.
- Its interface begins to unfold.
- Player discovers it was produced by the Corporation.

Stages:
- Stage 7.1: Pos.Aya logs anomaly.
- Stage 7.2: Pos.Aya notices item ownership change.
- Stage 7.3: PR Worm affects hidden layer.
- Stage 7.4: Folded interface opens.
- Stage 7.5: Corporate origin reveal seed.

Achievements:
- Anomaly Logged: trigger first unexpected log.
- Memory Route: Pos.Aya remembers a path.
- Folded Interface: open hidden panel.
- Witness Turn: Pos.Aya supports player choice against pure work enforcement.

Badges:
- Companion Badge: use Pos.Aya to choose recovery or agency.
- Hidden Architecture Badge: discover folded interface early.
- Corporate Tool Reversal Badge: use a corporate tool for human agency.

### Chapter 8. Portals and time return

Story:
- After M7, portals and time systems return with narrative meaning.
- Portals are not only transport; they reveal consequences, routes, and possible exit pathways.
- Pos.Aya interprets portal/time consequences.

Stages:
- Stage 8.1: Reactivate dedicated portal surfaces.
- Stage 8.2: Transport package while preserving state.
- Stage 8.3: Use portal to solve classification route.
- Stage 8.4: Use time/rift consequence to reveal story flag.
- Stage 8.5: Open route beyond compliance.

Achievements:
- Portal Valid: use a dedicated surface correctly.
- State Preserved: send item/package through portal without losing state.
- Time Insight: use time consequence to reveal story.
- Beyond Compliance: open a route that is not just productive.

Badges:
- Early Rift Badge: discover a time/portal affordance before prompted.
- ChronoRift Badge: combine portal and story consequence.
- Route Beyond Loop Badge: use portal/time to enable agency.

### Chapter 9. Life outside the loop

Story:
- The player removes the uniform.
- The player uses recovery spaces: exercise, meditation, reading, music, movement.
- Phone/social/family/project/travel routes become meaningful.
- Leaving the loop becomes success, not failure.

Stages:
- Stage 9.1: Remove uniform.
- Stage 9.2: Choose recovery area.
- Stage 9.3: Use phone or personal route.
- Stage 9.4: Decide toward family, project, travel, or meaningful exit.
- Stage 9.5: Final departure sequence.

Achievements:
- Uniform Off: remove corporate uniform.
- Recovery Practice: meditate, exercise, read, or rest.
- Contact Outside: use phone/social route.
- New Meaning: choose new project or life direction.
- Leave the Loop: complete exit arc.

Badges:
- Family Route Badge.
- New Project Badge.
- Travel Route Badge.
- Purpose Badge.
- Human Outside the Loop Badge.

## Achievement and badge rules

### Achievement

A specific completed action or objective.

Examples:
- send 5 boxes
- read first supervisor email
- open notebook
- place first block
- clear first jam

### Badge

A higher-level recognition of style, timing, sequence break, or combined achievements.

Badge triggers:
- doing something before expected
- beating a previous record
- completing several achievements in one run
- solving with fewer blocks
- completing without damage
- choosing recovery before collapse
- using a corporate tool for human agency

### Record badge

Awarded when the player improves personal or stage performance.

Examples:
- Fast Five
- Deadline Clear
- Minimal Route Record
- Fast Repair

### Sequence break badge

Awarded when the player discovers or completes a function before the tutorial route expects it.

Examples:
- opens portal surface early
- sorts by label before instruction
- uses Pos.Aya anomaly before prompt
- solves stage using two-box stack before taught

### Composite badge

Awarded when several achievements combine into a broader accomplishment.

Examples:
- Combo Shipping Badge
- Anti-Jam Specialist Badge
- Corporate Tool Reversal Badge

## Batch rules

- Packages arrive by batch.
- Start/New Batch button triggers receiving a batch.
- Every batch should be countable and visible.
- Shipment button sends valid packages currently in shipping area.
- Shipping area can support up to 20 boxes in a 5 x 2 x 2 arrangement.
- One push-button shipment can send the full valid shipping area.

## Button and portal placement rules

- Buttons can be positioned at opposite ends of the area.
- Wall buttons can exist beside the path.
- Walls near buttons can accept portal surfaces later.
- Portal surfaces can be above/below and on contiguous walls where valid.
- Portal placement must remain constrained to valid surfaces.

## Gizmo vision and action menus

Gizmo vision should show:
- jam alerts
- route status
- package state
- item ownership warnings
- portal validity
- surface context
- repair targets

Tapping an area or object opens a context menu with only relevant actions.
